# P8 UI Design

## Backend API Slice

The UI backend starts as a small FastAPI app over three injected data sources:

- Registry: model topology, SLOs, and TRS parameters.
- Service manager state client: `/v2/state` equivalent payload for the cluster grid.
- Redis hash client: `tre:v2:decision:latest` for the latest controller decision.

Implemented endpoints:

- `GET /healthz`
- `GET /api/cluster` returns registry topology plus service-manager state.
- `GET /api/models` returns model parameters for the parameter page.
- `GET /api/decision/latest` decodes the latest decision hash payload.
- `GET /api/experiments` is an explicit stub for the P8 experiment panel.

The frontend will use these endpoints without runtime external CDN dependencies.
